说明

base

网站核心公共样式，所有应用共用，专人维护，不能随意修改。

ui

网站全局公共样式，如按钮、提示等。所有应用共用，专人维护，不能随意修改。通过基本参数设置实现不同应用的定制。

component

网站业务组件类样式，如头部、导航、底部、分页等，根据不同应用编写，专人维护。

module

网站业务模块样式，模块样式层级和文件个数不限，但要遵循前端规范。较简单的模块，可以只有一个单独的css文件。对于较复杂的模块，如中文产品，可单独建立模块级公共样式和独立的页面级样式。

注意：新增样式文件要遵循此模块划分方法。一般维护，只需对module内文件进行增改。
